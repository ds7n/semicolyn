## Why "requires-attention", not "blocker"

Both are **usable** under our GPL-3.0-only license. Neither forces a relicense.
The "attention" is a set of **obligations we must satisfy**, not a compatibility
conflict:

1. **tldr-pages (CC-BY-4.0):**
   - CC-BY-4.0 → GPLv3 is one-way compatible per Creative Commons' documented
     compatibility position (verified at creativecommons.org). CC-BY (attribution
     only, NO ShareAlike) is strictly less restrictive than CC-BY-SA, so it flows
     into a GPLv3 work without a copyleft clash.
   - **Obligation: attribution.** CC-BY requires crediting the licensor, a license
     link, and indicating if changes were made. We heavily transform (extract
     command tokens, drop `{{placeholder}}` args, keep only frequency counts — a
     "plain frequency fingerprint" per the ingestion spec), which IS a change, so
     the "indicate changes" clause applies.

2. **Fig autocomplete (MIT):**
   - MIT is unconditionally GPL-compatible.
   - **Obligation: include the copyright + permission notice** in the distribution.

## What the shipped artifact actually contains (materiality)

The bundled seed (`seed_pinned.sketch`) is NOT verbatim corpus text. The pipeline
(`SeedKit/TldrParser` → `SeedBuilder`) extracts CLI command tokens (`git`, `commit`,
`--amend`), drops `{{placeholder}}` argument tokens, and stores a unigram + bigram
**frequency table** (token → count). Individual command names are uncopyrightable
facts; a transformed frequency fingerprint is a strong derivative-work argument for
fair reuse. Even so, the conservative and REUSE-consistent path is to carry
attribution rather than rely on the fingerprint being non-copyrightable.

## Required deliverables (fold into the seed-install spec)

1. **Attribution file** for the two corpora, e.g. `App/Resources/predictor/SEED_ATTRIBUTION.md`
   (bundled) or a NOTICE entry:
   - tldr-pages, CC-BY-4.0, link to license, "derived: command-token frequencies extracted; argument placeholders removed."
   - Fig autocomplete, MIT, copyright "© 2021 Hercules Labs Inc. (Fig)", full permission notice.
2. **Provenance record** written by the rebuild script alongside the blob
   (`seed_v1.provenance.txt`): the exact `TLDR_REF` + `FIG_REF` and resolved commit
   SHAs used (build-seed.sh already prints these), so the committed seed maps to an
   auditable corpus snapshot.
3. **REUSE compliance:** the committed binary seed blob needs a `.license` sidecar or
   REUSE.toml entry. Its own SPDX is our GPL-3.0-only (it is our build output), but
   the sidecar should note the upstream corpus licenses (CC-BY-4.0, MIT) it derives
   from, mirroring how the repo already excludes `.license` sidecars for fonts.
4. **In-app attribution surface:** ensure the CC-BY/MIT credits are reachable in a
   Settings → About / Acknowledgements screen (CC-BY attribution should be
   user-visible, not only in-repo).
