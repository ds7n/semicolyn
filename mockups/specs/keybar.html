Neotilde — Keybar v1 (locked layout)

The v1 keybar: locked-left default of Esc pill · Pad · Modifier · Tab, a customizable scroll region, and a unified Settings editor. Supersedes the v0 mockup at mockups/specs/keybar.html (which the spec itself flags as out-of-date) and the keybar passages in the context-detection spec. Esc pill — gesture detail

One pill, five distinct gestures. Tap is the Esc keystroke; horizontal swipes navigate windows; vertical swipes open the quick picker or create a window; long-press opens the unified picker (Live · Recent · Connect · Settings · Tips). See mockups/drafts/esc-pill.html for the full picker treatments.

Pill appearance

Label is the literal "Esc". The dim bronze "≡" is the Neotilde-wide "more gestures here" hint glyph. Pressed / picker-open state uses the bronze-tint fill — same treatment as armed modifiers.

Rest (left) and armed / pressed (right). Window title is not rendered on the pill — the user identifies the current window by terminal content or by opening a picker.

Pad — gesture detail

The Pad fuses arrows with pane management. Drag fires arrows; long-press arms pane-mode (bronze tint + ↕/↔ overlay); long-press + swipe commits a split. Tap toggles zoom on the active pane.

Gesture vocabulary

GestureAction
Drag (any dir)Arrow keystroke (↑ ↓ ← →). Immediate motion from touch-down.
TapZoom toggle on the active pane. Tap = touch + quick release without drag.
Long-press (~400ms)Arms pane mode — bronze tint + ↕/↔ overlay appears. Light haptic.
Long-press + swipe ↔Vertical split (tmux split-window -h).
Long-press + swipe ↕Horizontal split (tmux split-window -v).

Disambiguation: arrow drag requires immediate motion; pane mode requires holding without movement for ~400ms. Zoom state shows on the focused pane itself (bronze border + ⊕ in index badge), not on the Pad.

Reverse-bar — locked-left vs locked-right

A single Settings toggle flips the entire bar. Useful for left-thumb users or anyone whose grip favors the right edge. Pure layout mirror — gesture semantics on the Esc pill (swipe-left = previous window) do not flip.

Settings → Keybar editor

Single editable list. Drag handles reorder within a region; dragging across the dashed locked/scroll divider moves a slot between regions. Esc pill and Pad refuse the divider — a lock icon marks them. Swipe-left on a removable row reveals delete. The "+ Add" at the bottom opens the macro / custom-slot sub-sheet.

Mechanics

AffordanceBehavior
Drag handleReorder within current region. Drag across the divider moves between locked ↔ scroll.
🔒 lock iconEsc pill and Pad refuse divider crossings. Reorderable within locked, never movable to scroll, never removable.
Swipe-leftReveals Remove. Modifier shows one-time first-time warning ("You'll lose Ctrl/Alt/Shift access."). No nag on repeat.
+ AddSub-sheet: Pin a macro · Create new macro · Create new slot.
ResetRestores v1 default layout. Single tap, no confirm.

A user-created custom slot is a bundle of up to four bindings (tap, swipe-up, swipe-down, long-press), each pointing at a macro. Horizontal-swipe bindings are reserved for special widgets and deferred to v1.5+ for user slots.

Fn slot + F-key mode

Tapping Fn toggles F-key mode: the scroll region's contents cross-fade (~180ms) from the default symbols to F1–F12. Locked region is unchanged. Fn lives at the end of the scroll region by default — heavy users live in htop/top/mc where context detection auto-locks Fn for them.

Context-detection preview (compact)

When a known foreground process engages, its promotion set slides into the scroll region between the locked divider and the defaults. Promoted slots get a bronze-tint fill plus a thin top-edge accent. Cross-reference mockups/specs/context-detection.html for the full vim / python / less / psql story and the engage/disengage animation timing.
