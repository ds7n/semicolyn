Neotilde — Settings sub-screens

Three sub-screens whose tree positions were locked but whose contents weren't: Security, App preferences, and About & Help. Power-user knobs deferred. Every section narrow, every row earns its place.

Revises earlier decision

App lock framing flips. The prior credential-security note said "the user-facing biometric gate is app-level (Face ID once per session)."

This spec revises that: the user-facing gate is the device unlock. App-level Face ID is an opt-in extra layer, off by default. Notes / Mail / Messages don't gate themselves on an unlocked phone; Neotilde won't either by default. The anyUse per-identity policy remains the per-operation escape hatch for high-value keys.

4 — Cuts (explicit)

Things deliberately left out of these three sub-screens, with rationale:

  • Predictor pattern-exclude / retention / incognito review — power-user knobs; the defaults in the predictor spec carry v1.
  • Appearance / theme picker — one palette in v1.
  • Connection defaults under App prefs — covered by the Defaults editor reached from Settings → Hosts.
  • Forget-all-fingerprints — destructive blunt instrument; per-row swipe is the precise tool.
  • Search in any sub-screen — none are long enough to need it (under ~40 rows).
  • Terms of Service · Rate the app · Changelog — no service contract, friction-y prompts skipped, no v1.5 to write about yet.
  • iPad / external-keyboard layout — separate brainstorm.
  • Localisation — English only for v1.
