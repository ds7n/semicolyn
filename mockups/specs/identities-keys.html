Neotilde — Identities & Keys management surface

The standalone management screen for SSH identities. Reached via Settings → Identities & Keys or by tapping an identity reference inside host CRUD. Colors and components come from design-system.html; sub-flow is shared with host-crud.html.

List screen

Alphabetical by display name. Flavor chip (iCloud / SE) on the right; usage indicator below it. No swipe-actions: delete lives only on the detail screen, behind a confirmation sheet.

Locked decisions

  • Scope: inventory + standalone create. No rotation wizard in v1 (deferred until ssh-copy-id auto-install lands in v1.5).
  • List sort: alphabetical by display name. No filter, no search, no sort toggle.
  • Row anatomy: name + algorithm + fingerprint-prefix + flavor chip + usage count. Whole row taps to detail. No swipe actions.
  • Empty state: centered icon, one-line copy, single CTA opening the create/import half-sheet.
  • Detail edits: display name (inline), auth policy (3-segment control). Both commit immediately, no Save button.
  • Public key / fingerprint: both have inline Copy actions. No QR, no share-sheet in v1.
  • "Used by" drill-down: full-screen push to host list (same component as Settings → Hosts, filtered to this identity).
  • Delete (unreferenced): bottom-anchored destructive button → action sheet from bottom. Confirm row positioned upward from original button; Cancel is closest to thumb. Copy varies by flavor.
  • Delete (referenced): refusal pattern matches host CRUD jumphost delete. Surface back-references, no destructive option offered.
  • Auth policy change: plain toggle, no biometric. App-level unlock is the security gate (matches Blink / Termius / 1Password / Secretive convention).
  • Import sources: paste + Files app picker. No Share Extension in v1.
  • Import flavor: always iCloud Keychain. SE cannot accept external key material.
  • Entry points: Settings → Identities & Keys, plus tappable identity refs from host CRUD.
